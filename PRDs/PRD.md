# 📝 电脑综合服务平台 (PC-Service Hub) 产品需求文档

## 1. 项目概述
* **目标**：构建一个极简、专业的电脑维修/维护预约网页。
* **用户路径**：选择服务 -> 录入型号 -> 填写地址/电话 -> 选择时间 -> 提交成功。
* **管理路径**：管理员通过 `/admin` 路径查看并管理所有预约订单。
* **技术栈建议**：Next.js (App Router), Tailwind CSS, Lucide React (图标), Shadcn UI (组件库)。

---

## 2. 核心功能规范

### 2.1 用户端：分步预约表单 (Multi-step Form)
无需登录，采用进度条式交互，确保用户无压力完成填单。

| 步骤 | 模块名称 | 关键字段/交互 | 细节要求 |
| :--- | :--- | :--- | :--- |
| **Step 1** | **服务选择** | 清灰换硅脂、系统重装、数据恢复、硬件升级、系统优化 | 卡片式布局，点击选中高亮，支持单选。 |
| **Step 2** | **设备信息** | 设备类型（笔记本/台式机/一体机）、品牌型号、问题描述 | 型号建议设为必填，问题描述设为选填。 |
| **Step 3** | **服务方式** | 上门服务 / 预约到店、联系电话、详细地址 | 手机号需进行 11 位数字格式校验。 |
| **Step 4** | **时间预约** | 预约日期（Date）、服务时段（Time Slot） | 禁用历史日期；时段推荐：09:00-12:00, 13:00-18:00, 19:00-22:00。 |

### 2.2 管理端：订单管理后台 (Admin Dashboard)
* **入口**：`/admin`（建议设置简单的环境变量密码访问）。
* **功能**：
    * **订单列表**：以表格（Table）形式展示所有数据。
    * **状态切换**：每个订单操作栏有“待处理”、“已完成”、“取消”按钮。
    * **数据持久化**：前期可使用 `localStorage` 或简单的 `JSON file`，后期可对接 Supabase/Prisma。

### 2.3 支付说明
* **结算方式**：**纯线下付款**。
* **流程**：用户提交订单后显示“预约成功”，提示“工程师将与您联系，服务完成后线下结算”。

## 3. UI/UX 视觉规范
* **风格**：极简科技感（类似 Apple Support 或华为服务官网）。
* **色调**：
    * 主色：`Blue-600` (#2563eb) - 代表专业与科技。
    * 背景：`Slate-50` (#f8fafc) - 保持清爽。
* **组件建议**：
    * 使用 **Stepper** 组件展示当前进度。
    * 按钮在点击后应有 Loading 状态，防止重复提交。

---

## 4. 数据库字段定义 (Schema)
如果使用数据库（或 Mock 数据），字段如下：

```typescript
interface Order {
  id: string;               // 订单号：PC-YYYYMMDD-随机数
  service: string;          // 服务项目
  deviceType: string;       // 笔记本/台式机
  model: string;            // 具体型号
  locationType: 'onsite' | 'store'; // 上门 / 到店
  address: string;          // 地址
  phone: string;            // 手机号
  appointmentDate: string;  // 预约日期
  appointmentTime: string;  // 预约时段
  status: 'pending' | 'success' | 'cancelled'; // 订单状态
  createdAt: number;        // 创建时间戳
}
```

## 5. 给 Cursor 的开发 Prompt (可直接复制)

> **指令：**
> 请根据以下需求开发一个 React/Next.js 应用：
> 1. **首页布局**：创建一个全屏的分步预约表单（Stepper）。
> 2. **表单逻辑**：
>    - 第一步：提供 5 个服务选项（清灰、系统、解码、升级、其他）。
>    - 第二步：选择设备类型（下拉）并输入品牌型号。
>    - 第三步：输入 11 位手机号和详细地址。
>    - 第四步：使用日期选择器和时段选择（上午/下午/晚上）。
> 3. **提交逻辑**：点击提交后，将数据保存至本地存储（localStorage），并跳转至“成功页面”。
> 4. **后台页面**：在 `/admin` 路径下创建一个管理表格，能够看到所有订单，并可以点击按钮修改订单状态为“已完成”。
> 5. **设计风格**：使用 Tailwind CSS。要求响应式设计（移动端适配），配色采用深蓝色调，整体视觉效果要专业、简洁。
